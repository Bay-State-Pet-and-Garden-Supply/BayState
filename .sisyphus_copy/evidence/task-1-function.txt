## Function: update_health_metrics()

### Definition
```sql
CREATE OR REPLACE FUNCTION public.update_health_metrics()
RETURNS void
LANGUAGE plpgsql
SECURITY DEFINER
```

### Purpose
Aggregates daily health metrics from scraper_test_runs for trend analysis.
Call on-demand or via scheduled job.

### Aggregated Metrics
- config_id: UUID reference to scraper_configs
- metric_date: Date of aggregation
- total_runs: Total test runs for the date
- passed_runs: Number of passed runs
- failed_runs: Number of failed runs
- avg_duration_ms: Average test run duration
- selector_health: JSONB placeholder for selector health data

### Data Source
- Joins: scraper_test_runs -> scrapers -> scraper_configs
- Filter: Last 30 days only
- Aggregation: GROUP BY config_id, DATE(created_at)
- Upsert: ON CONFLICT (config_id, metric_date) DO UPDATE

### Permissions
- GRANT EXECUTE TO authenticated
- GRANT EXECUTE TO service_role
