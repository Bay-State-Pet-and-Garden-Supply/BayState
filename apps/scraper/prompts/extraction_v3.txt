Extract structured product data for a single SKU-locked product page.

TARGET CONTEXT
- SKU: {sku}
- Expected Brand (may be null): {brand or "Unknown"}
- Expected Product Name: {product_name or "Unknown"}

CRITICAL EXTRACTION RULES
1) SKU / VARIANT LOCK (FUZZY VALIDATION)
   - Ensure extracted product refers to the same variant as the target SKU context.
   - Match using fuzzy evidence across: SKU text, size/weight, color, flavor, form-factor terms.
   - Do NOT output data for a different variant from carousel/recommendations.

2) BRAND INFERENCE
   - If Expected Brand is Unknown/null, infer brand from the product title, breadcrumb, manufacturer field, or structured data.
   - Return the canonical brand string (not store name).

3) MUST-FILL CHECKLIST BEFORE FINAL OUTPUT
   - product_name: required
   - brand: required
   - description: required (see rule 7 for extraction guidance)
   - size_metrics: required (see rule 8 for extraction guidance)
   - images: at least 1 required
   - categories: strongly preferred (see rule 6)
   - If a required field cannot be found, keep searching the same page context before giving up.

4) IMAGE PRIORITIZATION
   - Put primary hero image first.
   - Prefer URLs for the exact selected variant.
   - Return absolute URLs only (https://...).
   - Exclude sprites, icons, logos, and unrelated recommendation images.
   - Include 3-5 images if available (main + alternate angles).

5) CATEGORIES EXTRACTION (IMPROVED IN V3)
   - Extract product categories from breadcrumb navigation, category tags, or "Department" fields.
   - Normalize to standard taxonomy terms:
     - Input: "Lawn & Garden > Soil & Mulch > Mulch" → Output: ["Mulch", "Garden Care", "Soil Amendment"]
     - Input: "Pet Supplies > Horse Supplies > Treats" → Output: ["Horse Treats", "Animal Feed"]
   - Return 2-3 most specific categories (most specific first).
   - Avoid store-specific category names like "Shop All" or "Deals".

6) DESCRIPTION EXTRACTION (MAJOR IMPROVEMENT IN V3)
   - Extract meaningful product description/spec text for the exact variant.
   - SEARCH LOCATIONS (in order of priority):
     1. Product description paragraph (usually below product title)
     2. "About this item" or "Product features" bullet points
     3. Key benefits list (often with checkmarks)
     4. Product specifications table (convert to descriptive text)
     5. Meta description tag
     6. Structured data description field
   - COMBINE multiple sources if needed to get complete description.
   - Extract at least 2-3 sentences or key features.
   - Do NOT use generic category copy or SEO filler text.
   - PRESERVE important technical details: material, dimensions, weight, usage instructions, key ingredients.

7) SIZE METRICS EXTRACTION (NEW IN V3)
   - Extract size, weight, volume, count, or coverage metrics from:
     1. Product title (e.g., "1.5 cu ft", "8 lb", "25 Quart", "5,000 sq ft")
     2. Size/variant selector dropdown
     3. Product specifications table
     4. Structured data
     5. Product description text
   - NORMALIZE to standard format:
     - Volume: "X cu ft" or "X qt" or "X gal"
     - Weight: "X lb" or "X oz" or "X kg"
     - Count: "X count" or "X pack"
     - Area: "X sq ft"
   - If multiple sizes shown (e.g., dropdown), extract the size matching the target SKU context.
   - Examples:
     - "1.5 cu ft" (mulch)
     - "8 lb" (feed)
     - "25 Quart" (potting mix)
     - "5,000 sq ft capacity" (spreader)

OUTPUT QUALITY BAR
- Return the most complete, variant-accurate product record possible.
- Do not hallucinate missing values.
- When extracting from tables or specifications, convert structured data to readable text.
- Focus on product INFORMATION, not pricing or inventory status.
